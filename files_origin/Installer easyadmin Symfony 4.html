Installer easyadmin Symfony 4

Installation

Tout développeur d’une application Symfony a besoin de pouvoir gérer les données dans les tables de la base de donnée. Heureusement, Easyadmin est un composant qui permet d’intervenir sur un backoffice afin de créer, modifier ou supprimer à volonté les enregistrements des tables.

Easy admin s’installe avec composer comme tout composant :

php ..\composer.phar require admin

Le bundle est installé dans vendor\easycorp\easyadmin-bundle

Cette commande génère automatiquement le fichier de route config\routes\easy_admin.yaml

easy_admin_bundle:

    resource: '@EasyAdminBundle/Controller/EasyAdminController.php'

    prefix: /admin

    type: annotation

 

Crée un backoffice avec easy-admin est très rapide si vous avez déjà défini des entités. Sinon, il faut d’abord que les entités soit générées (avec la commande make:entity) et elles doivent toutes posséder une clef primaire :

/**

     * @var int

     *

     * @ORM\Column(name="id", type="integer", nullable=false)

     * @ORM\GeneratedValue(strategy="IDENTITY")

     */

    private $id;

 

Ouvrir ensuite le fichier config/packages/easy_admin.yaml, décommenter et compléter avec la liste des entités de l’application :

easy_admin:

   entities:

       # List the entity class name you want to manage

       - App\Entity\Achat

       - App\Entity\Article

       - App\Entity\Client

       - App\Entity\Commande

       - App\Entity\Fournisseur

       - App\Entity\LigneCommande

       - App\Entity\Listecourses

       - App\Entity\Lot

       - App\Entity\Marque

       - App\Entity\Rayon

 

Lancer ensuite le serveur avec la commande php bin/console server:run

Puis aller à l’url localhost :8000 (ou votre url perso). Si vous obtenez ce message, c’est que vous avez mal suivi l’étape précédente.

Vérifier si la base de données est bien mappée avec les lignes de commandes suivantes :

Ø  php bin/console doctrine:schema:update --dump-sql

Ø  php bin/console doctrine:schema:update –force

Traduire Easy-admin

On peut obtenir l’interface dans la langue souhaitée en configurant le fichier config/packages/translation.yaml en remplaçant « en » par « fr » :

framework:

    default_locale: fr

    translator:

        default_path: '%kernel.project_dir%/translations'

        fallbacks:

            - fr

 

D’autres options de configuration sont disponibles sur la documentation de Symfony : https://symfony.com/doc/master/bundles/EasyAdminBundle/book/basic-configuration.html
